      # dummy节点的使用

      为什么需要dummy节点？

      第一种情况，将dummy作为新链表的头部节点。

      例题：21 https://leetcode-cn.com/problems/merge-two-sorted-lists/

      例题：2 https://leetcode-cn.com/problems/add-two-numbers/

      可以减少重复的代码逻辑，很灵巧的解决链表头部不好赋值的问题。

      第二种情况，解决链表头部的极端情况

      例题：19 https://leetcode-cn.com/problems/remove-nth-node-from-end-of-list/

      如果链表一共有n个节点，然后需要删除倒数第n个节点。那就意味着，链表的第一个节点肯定要被删除。

      所以我们定义一个dummy节点，作为链表的头节点，这样就不用去对第一个节点被删除再处理一遍逻辑了。

      # 递归的使用

      例题：24 https://leetcode-cn.com/problems/swap-nodes-in-pairs/

      递归的方法，结合dummy的虚拟节点

      	// 将head和head.next进行替换
function swap(prev, head) {
  if（head == null || head.next == null ){
    return;
  }
  let cnext = head.next;
  // 将head.next 重新指向
  head.next = cnext.next;
  cnext.next = head;
  prev.next = cnext;
  
  // 递归
  prev = head;
  head = head.next;
  swap(prev, head);
}

      终止条件

      head 为空指针或者 next 为空指针，也就是当前无节点或者只有一个节点，无法进行交换

      总结

      这个算法题，一开始，我没考虑到递归的算法。

      我考虑的是用一个临时变量 i来存储链表循环的次数。

      如果i是奇数的话，我认为达到了需要替换的时候。然后将奇数前一个节点进行替换。

      看了别人的解答，发现实际上这个i是多余的，因为我就直接从0开始替换就好了。将head和head>next替换完，然后去替换下一head和head>next;
