    它还可以帮助开发做静态代码的检测。

    # interface接口

    可选属性： 在属性后放一个？关键字

    readonly属性： 在属性前放一个readonly关键字

    interface SquareConfig {
	color?: number,
	height: number,
  readonly width: number,
  readonly height: number,
  [propName: string]: any, // 可索引的类型 表示接口里可以有任意类型的其他属性
  (name:string): boolean,  // 函数类型的属性，参数列表里的每个参数都需要名字和类型。

}

    索性类型

    interface StringArray {
   [index: number]: string, //索引类型
}

let myArray: StringArray; // StringArray的index只能是数字
myArray = ["Bob", "Fred"]; // StringArray[0], StringArray[1]

    interface可以定义一个类的接口，被class继承

    interface ClockInterface{
  tick();
}
// 类实现接口
class Clock implements ClockInterface {
  constructor(h: number, m: number) { } // constructor属于静态属性
  tick() {
    console.log("beep beep");
  }
}

    要注意的地方：==ts只对class的实例部分进行类型检查==。

    下面因为当一个类实现了一个接口时，只对其实例部分进行类型检查。 constructor存在于类的静态部分，所以不在检查的范围内。

    interface ClockConstructor {
    new (hour: number, minute: number);
}

class Clock implements ClockConstructor { // 报错，因为没有new 方法
    currentTime: Date;
    constructor(h: number, m: number) { } 
}

    interface可以被继承

    interface Shape {
    color: string;
}

interface PenStroke {
    penWidth: number;
}

interface Square extends Shape, PenStroke {
    sideLength: number;
}

    interface还可以继承类

    class Control {
    private state: any;
}

interface SelectableControl extends Control {
    select(): void;
}
