        # Nuxt

        学会服务端渲染的使用、部署流程。

        2种部署方式：1种静态部署、1种server部署

        服务端渲染

        # nuxt命令

        nuxt自带4个命令：

        "scripts": {
    "dev": "nuxt",
    "build": "nuxt build",
    "start": "nuxt start",
    "generate": "nuxt generate"
  },

        nuxt主要是用来开启一个开发环境

        nuxt build用来打包生产环境需要的文件，配合nuxt start来开启一个服务端渲染的server。

        如果是部署服务器，build之后需要把一些文件拷贝到服务器上去才能运行start，这个后面会讲到。

        nuxt generate会生成一个可以直接部署使用的静态的资源，它是不涉及服务端渲染的。需要配合nginx服务器来部署

        可以理解为nuxt generate只是利用webpack打包生成了静态文件。

        # Nuxt部署生产环境

        1.npm run build => nuxt build 生成文件

        2.在服务器上传.nuxt, express,static,nuxt.config.js,package.json文件，.nuxt里包含了执行所需要的文件

        3.在服务器执行npm install && npm run start启动一个服务，访问程序，看看是否有问题。

        4.利用pm2来跑后台守护进程

        # Nuxt里使用SVG图片

        安装依赖： @nuxtjs/svg

        配置nuxt.js:

        引入svg图片，类型为raw，这样svg图片会被认为是html代码处理。

        import HBLogo from "~/assets/Huobi.svg?raw"; 

export default {
  name: "Investors",
  data() {
    return {    
      imgList: [
        HBLogo,
      ]
    }
  }
}

        在html里使用svg图片，作为html标签代码

        <div class="imgDiv" v-for="(item, index) of imgList" :key="index">
  <div v-html="item" class="svg"></div>
</div>
