    # output

    output选项指定webpack输出的位置，其中比较重要的也是经常用到的有path和publicPath

    # output.path

    • 默认值：process.cwd()

    output.path只是指示输出的目录，对应一个绝对路径，例如在项目中通常会做如下配置：

    output: {
	path: path.resolve(__dirname, '../dist'),
}

    # output.publicPath

    • 默认值：空字符串

    publicPath可以指定项目里其他资源（图片、css、js）的加载路径。

    # 静态资源最终访问路径 = output.publicPath + 资源loader或插件等配置路径

    比如

    1、output.publicPath + filename

    // process.env.PUBLIC_URL: https://static.bhfastime.com/bluehelix/
output: {
    path: path.resolve(__dirname, "./dist"),
    publicPath: `${process.env.PUBLIC_URL || '/'}`,
    filename: "[name].bundle.js",
},

// 最终js路径为： https://static.bhfastime.com/bluehelix/[name].bundle.js

    2、output.publicPath + url-loader配置

    // process.env.PUBLIC_URL: https://static.bhfastime.com/bluehelix/
output: {
    publicPath: `${process.env.PUBLIC_URL || '/'}`,
},
  
module: {
  rules: [
  	{
        test: /\.(png|jpe?g)$/,
        loader: "url-loader",
        options: {
          limit: 1024,
          name: "[name].[hash:8].[ext]",
          outputPath: "assets/",   //输出图片放置的位置
          // publicPath: '/', // 默认就是webpack.output.publicPath + outputPath,
        },
      },
  ]
}

// 如果url-loader没配置publicPath, 最终图片的路径为:https://static.bhfastime.com/bluehelix/assets/[name].[hash:8].[ext]
// 如果url-loader配置了publicPath = /, 则最终路径为 /[name].[hash:8].[ext]

    3、output.publicPath + ExtractTextPlugin.filename

    // process.env.PUBLIC_URL: https://static.bhfastime.com/bluehelix/
output: {
    publicPath: `${process.env.PUBLIC_URL || '/'}`,
},
plugins: [
  // extract-text-webpack-plugin css
  new ExtractTextPlugin({
    filename: 'style.[chunkhash].css'
  })
]
// 最终css的访问路径为
// output.publicPath + 'style.[chunkhash].css' = '/dist/style.[chunkhash].css'

    # webpackDevServer

    在开发阶段，我们借用devServer启动一个开发服务器进行开发，这里也会配置一个publicPath，这里的publicPath路径下==的打包文件==可以在浏览器中访问。

    而静态资源仍然使用output.publicPath。

    webpack-dev-server打包的内容是放在内存中的，这些打包后的资源对外的的根目录就是publicPath。

    比如我们启动了一个webpack-dev-server 端口是8081:

    我们访问 http://localhost:8081/webpack-dev-server 可以来看webpack-dev-server打包的目录

    点击目录里的资源链接，比如list.html, 它的路径就是：

    http://localhost:8081/${publicPath}news/list.html

    其中${publicPath}是webpack-dev-server里配置的publicPath.

    如果不配置，则返回``http://localhost:8081/undefinednews/list.html`

    一般情况下都要保证devServer中的publicPath与output.publicPath保持一致。
