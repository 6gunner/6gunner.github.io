      # shouldComponentUpdate VS React.PureComponent

      shouldComponentUpdate此方法仅作为性能优化的方式而存在。不要企图依靠此方法来“阻止”渲染，因为这可能会产生 bug。

      如果你一定要手动编写此函数，可以将 this.props 与 nextProps 以及 this.state 与nextState 进行比较，并返回 false 以告知 React 可以跳过更新。请注意，返回 false 并不会阻止子组件在 state 更改时重新渲染。 // todo 测试一下，子组件怎么更新的

      我们不建议在 shouldComponentUpdate() 中进行深层比较或使用 JSON.stringify()。这样非常影响效率，且会损害性能。

      PureComponent 会对 props 和 state 进行浅层比较，并减少了跳过必要更新的可能性。

      如果对象中包含复杂的数据结构，则有可能因为无法检查深层的差别，产生错误的比对结果。对于==复杂数据结构的props==不要用这个组件，仅在你的 props 和 state 较为简单时，才使用 React.PureComponent，或者在==深层数据结构发生变化==时调用 forceUpdate() 来确保组件被正确地更新。

      # React.memo

      如果你的组件在相同的props情况下，会返回相同的结果。那么可以利用React.memo来包装组件。

      React.memo是一个高阶组件，可以传入一个组件，memo会props是否发生变化，以此通过记忆组件渲染结果的方式来提高组件的性能表现。

      # 怎么在react里使用throttle ？

      一般能想到的就是在constructor里用throttle或者debounce来包裹函数

      但是如果用到了event，就会发现event.target是null。

      这时候，就需要把内部函数提取出来，用throttle或者debounce包装，而不能把onChange事件包装起来。

      错误代码：

      class ModalSetting extends React.Component {
  constructor(props) {
    super(props);
    this.state = {
      quickCloseConfirm: props.order_setting.orderS,
    };
    this.handleQuickCloseConfirmChange = throttle(
      this.handleQuickCloseConfirmChange,
      500
    );
  }

// 启用/禁用闪电平仓弹窗
  handleQuickCloseConfirmChange = (event) => {
    this.setState(
      {
        quickCloseConfirm: event.target.checked ? 1 : 0,
      },
      () => {
        // todo
        console.log("调用接口");
      }
    );
  };
}

      正确代码：

      class ModalSetting extends React.Component {

	constructor(props) {
    super(props);
    this.state = {
      quickCloseConfirm: 0,
    };
    this.setQuickCloseConfirm = debounce(this.setQuickCloseConfirm, 100);
  }
  
   // 启用/禁用闪电平仓弹窗
  handleQuickCloseConfirmChange = (event) => {
    const quickCloseConfirm = event.target.checked ? 1 : 0;
    this.setQuickCloseConfirm(quickCloseConfirm);
  };

	// 提取出内部的业务
  setQuickCloseConfirm = (val) => {
    this.setState(
      {
        quickCloseConfirm: val,
      },
      () => {
        // todo
        console.log("调用接口");
      }
    );
  };
  
}
