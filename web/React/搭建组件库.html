# 3、组件库的颜色体系

​ 一般一个系统的颜色体系有两块：基础色、中性色。

​ 基础色就是系统的主色调，主色调又可以生成对应的衍生色。像antd-design里提供了12个主色以及对应的衍生色，可以选择。

​

​ 我们设计师说🐜的这套，一般用6号色作为基础色，然后对应的按钮的悬浮、点击的颜色，相差一位色调就可以。

​ 中性色是用来一般用来定义文字的颜色

​

# 4、定义组件库的变量

组件库的变量需要进行分类

  • 基础色彩系统

  • 字体系统(font-family、字体大小、字重、行高、标题大小、颜色)

  • 表单

  • 按钮

  • 边框、阴影（修饰类）

看看蚂蚁ant-design的结构设计，他们的样式代码库非常有条理

  • color 里定义了一些颜色，和他们设计的没人能看懂的算法，这些算法可以去轻微修改主色的色调，形成一个色调版。

  • core 定义了很多基础的、通用的样式，并且做了浏览器的兼容，确保各个浏览器样式一致。

    • motion 定义了一些动画效果keyframes

  • mixins 定义了一些混入方法 主要用来在组件里，通过一些变量，来修改一些样式，减少代码重复。

  • themes就是主题样式

@mixin btn-size($padding, $fontSize, $height) {
  padding: $padding;
  font-size: $fontSize;
  height: $height;
}

@mixin btn-style(
  $bgColor,
  $borderColor,
  $color,
  $hoverBgColor: lighten($bgColor, 7.5%),
  $hoverBorderColor: $borderColor,
  $hoverColor: $color
) {
  color: $color;
  background: $bgColor;
  border-color: $borderColor;
  &:hover {
    color: $hoverColor;
    background: $hoverBgColor;
    border-color: $hoverBorderColor;
  }
  &:focus,
  &.focus {
    color: $hoverColor;
    background: $hoverBgColor;
    border-color: $hoverBorderColor;
  }
  &.active,
  &:active {
    outline: 0;
    -webkit-box-shadow: none;
    box-shadow: none;
  }
}

# 6、测试react组件库

单元测试是组件库里很重要的一部分。

除了使用jest这一个通用的测试库之外，react组件的单元测试还需要用到一些专门的测试lib。

react官网给我们推荐了@testing-library/react这一个库。

地址：https://github.com/testing-library/react-testing-library#basic-example

它还有一系列的断言插件库，帮助写测试用例

"@testing-library/jest-dom": "^4.2.4",
"@testing-library/react": "^9.3.2",
"@testing-library/user-event": "^7.1.2"

react这个库里提供了render, fireEvent等方法

Jest-dom提供了很多api用来帮助dom的判断：https://github.com/testing-library/jest-dom

jest这个库提供了一些通用的方法：https://jestjs.io/docs/en/mock-functions

describe('Button Test', () => {
 it('should render normal btn', () => {
  const onClick = jest.fn(); // 定义一个mock方法
  const document = render(<Button data-testid="button" onClick={onClick}>Test</Button>); // 拿到一个dom
  const btn = document.getByTestId('button'); // 获取到组件
  expect(btn).toHaveClass('btn', 'ez-btn'); //判断class
  const element = document.getByText('Test'); 
  expect(element.textContent).toEqual('Test'); //判断文本
  fireEvent.click(btn); 
  expect(onClick.mock.calls.length).toBe(1); // 判断事件的触发
 });
});

# 组件间通信

因为涉及到多个组件，当菜单点击时，事件需要从item里传递给父组件，来判断哪一个菜单被点击了，从而计算出选中的菜单，以及展开的菜单样式。

组件之间传递可以通过Provider.Context 或者 利用mini-store来实现。

# React,key

key作为react的预留属性，是不会在props里被读取到的。

如果想要用key这个Props，需要在组件编译前，在内部将key复制给其他的props。

比如，我定义了一个Menu组件，我想用key作为menuItem的唯一主键，所以我需要在组件内部，将key拿到，然后eventKey来接收，传入到子组件里。
