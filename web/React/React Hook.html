                    # React Hooks

                    官方文档

                    react hook是react在16.8版本后推出的新特性。 用来解决在函数式组件里，需要用到类似组件里state、生命周期函数等功能的问题。

                    react hook vs 组件生命周期函数的区别

                    在类组件中，我们经常将有相关业务逻辑的方法拆分到不同的生命周期中； 同时，又经常把一些无相关性逻辑的代码写到同一个生命周期中。 虽然这是我们已经默认接受了的写法，但是这依然造成了代码的连贯性和可阅读性很低。

                    但是使用hooks的话，可以在useEffecthook里将逻辑拆分组合起来，可以很好的解决以上的问题。

                    # useState

                    useState接收参数，作为初始化值，返回一个包含了[state, setState]的数组。

                    const [state, setState] = useState(initialState);

                    函数参数用法1

                    setState接收的参数不仅仅可以是一个普通对象object。也可以是一个函数function。

                    最常见的，传入一个函数，接收当前state参数，来对现有的state进行递增、递减的运算。

                    函数参数用法2

                    在useState时，同样也可以接收一个函数参数。如果我们的初始化initialState是高开销的计算结果，可以改为提供函数。这个函数仅仅在初始渲染时执行，不用担心占用了过多的资源（cpu计算）。

                    const [state, setState] = useState(() => {
  const initialState = someExpensiveComputation(props);
  return initialState;
});

                    setState的重新渲染机制

                    在调用setState方法，更新状态后，会触发组件的重新渲染。

                    但是调用setState==并不会立即重新渲染==。 React会将多次setState的调用合并，在一次调用中对组件进行重新渲染，展示更新。

                    # useEffect

                    useEffect接收两个参数，第1个参数是一个函数，当组件重新渲染时执行；第2个参数是可选参数，指明effect的依赖项。如果依赖项不变，传入的第1个函数不会执行。

                    useEffect(() => {
  document.title = title
}, [title]);

                    useEffect可以返回一个函数，这样的话，useEffect会在执行函数之前，先执行上一次返回的函数。

                    useEffect(() => {
	console.log('开启定时器');
	let $timer = setInterval(() => {
		setNumber(number => number + 1);
	}, 1000)
  // useEffect如果返回一个函数，该函数会在组件卸载和更新时调用。
  // react在执行effect函数之前，会先调用上一次返回的函数。
	return () => {
		console.log('销毁定时器');
		clearInterval($timer);
	}
}, [number]);

                    useEffect的几个要点

                    useEffect会在组件渲染到屏幕以后，执行effect函数。且每次组件重新渲染后，会再次执行。

                    如果不希望useEffect的函数重复执行，可以接收的第2个数组参数里，添加依赖属性。那样只有当属性发生变化时，effect函数才会被执行。

                    如果传入空数组[]，意味着这个effect函数只会在组件第一次mount时执行一次。但是不要把这个等同为mounted生命周期函数，因为useEffect这个钩子实际上每次都会执行。

                    vs useLayoutEffect的区别

                    1、执行机制上

                    useLayoutEffect会先于useEffect执行。

                    文档上说，useLayoutEffect会在所有的 DOM 变更之后==同步==调用 effect。可以使用它来读取 DOM 布局，并同步触发重渲染。在浏览器执行绘制之前，useLayoutEffect 内部的更新计划将被同步刷新。

                    这里的同步的概念，可以理解为组件render后，立即调用方法。对于useEffect的异步，是利用useIdleRequest来执行的。

                    2、组件生命周期上

                    假如有以下组件：

                    function Test3() {
  const [count, setCount] = useState(0);
  var a = useRef("xxx");
  useEffect(() => {
    console.log(a, "useEffect");
    document.title = `You clicked ${count} times`;
    return () => {
      console.log(a, "end useEffect");
      document.title = `remove`;
    };
  });
  useLayoutEffect(() => {
    console.log(a, "useLayoutEffect");
    document.title = `You clicked ${count} times`;
    return () => {
      console.log(a, "end useLayoutEffect");
      document.title += "!!!";
    };
  });
  console.log(count, "更新Example");
  return (
    <div>
      <p>You clicked {count} times</p>
      <button onClick={() => setCount(count + 1)}>Click me</button>
      <Child></Child>
    </div>
  );
}

class Child extends React.Component {
  componentDidMount() {
    console.log("Child mount");
  }
  componentDidUpdate() {
    console.log("Child update");
  }
  render() {
    return <span>Child</span>;
  }
}

                    第一次渲染时，子组件先渲染，然后useLayoutEffect，再执行useEffect.

                    组件更新时，会先执行useLayoutEffect返回的函数，然后update子组件，再执行useLayoutEffect函数。

                    最后才执行useEffect返回的函数和useEffect传入的方法

                    # useReducer

                    当setState的变化逻辑很复杂时，可以使用useReducer来很清晰的实现逻辑

                    使用reducer可以减少函数里对state的依赖。

                    import React from "react";
import { useReducer } from "react";

function TestUseReducerPage() {
	// useReducer接收2个参数，一个reducer函数，一个初始对象。
	// action可以是任意类型，初始对象也可以是任意类型
  const [number, dispatch] = useReducer((state, action) => {
    switch (action.type) {
      case "add1":
        return state + 1;
      case "add2":
        return state + 2;
      default:
        return state;
    }
  }, 1);

  const handleClick = (type) => {
    dispatch({
      type,
    });
  };

  return (
    <div>
      {number}
      <button onClick={(e) => handleClick("add1")}>+1</button>
      <button onClick={(e) => handleClick("add2")}>+2</button>
    </div>
  );
}

export default TestUseReducerPage;

                    # createContext & useContext

                    createContext创建一个上下文context，useContext来使用上下文context。两个结合起来使用，来避免在多层组件间嵌套传递props造成代码臃肿。

                    上层组件

                    const themes = {
  light: {
    foreground: "#000000",
    background: "#eeeeee",
  },
  dark: {
    foreground: "#ffffff",
    background: "#222222",
  },
};

export const ThemeContext = React.createContext(themes.light);

function App() {
  const theme = React.useContext(ThemeContext);
  console.log(theme); // 如果useContext不在Context.Provider里被调用，它会返回一个默认值
  return (
    // 我们将 “dark” 作为当前的值传递下去。
    <ThemeContext.Provider value={themes.dark}>
    	...
    </ThemeContext.Provider>
  )
}

                    子组件

                    import React, { useContext } from "react";

import { ThemeContext } from "../../App";

function TestUseContext() {
  return (

  const theme = useContext(ThemeContext);
  return <h1 style={theme}> 我是child，我被useContext样式控制了</h1>;
}

                    # useRef vs createRef

                    相同点

                    两者都可以创建一个mutable对象，用来保存临时的对象，这个对象可以是dom引用，也可以是普通的js对象。

                    区别

                    useRef是react的钩子函数，只能用在Function Component里。

                    createRef一般只用在组件函数里。原因是：

                    • createRef每次执行时，不会对上一次的ref值进行保留。如果createRef用在function component里，每次组件重新渲染时，都会被重新执行。导致ref值丢失，取不到ref.current的值。
                    • useRef只在组件初始化（第一次被调用）时被赋予初始值，之后组件重新渲染调用该方法，值依然保持不变。

                    所以createRef一般都在React Component的初始化生命周期里执行。比如：constrcutor、componentWillMount、

                    useRef在Function Component里使用，你可以把它的作用理解为：将第n帧的props或者state保存在ref的current对象上，然后在第n+1帧里去读取ref的值。

                    useRef的使用方法

                    function App {
    const divRef = React.useRef() // 存储dom
    const valueRef = React.useRef(90) // 存储普通number
    const [,setDummyState] = useState()
    return (
        <div>
            Value: {valueRef.current}
            <div id="divR" ref={divRef}>App, here</div>
            <button onClick={()=> (valueRef.current = 88, setDummyState({}))}>Incr</button>
        </div>
    )
}

                    利用useRef来避免闭包问题

                    function Timer() {
	const [count, setCount] = useState(0);
	const ref = useRef();
	ref.current = count; // 每次count变化时，都会重新给ref赋值
	const handleClick = () => {
		setTimeout(() => {
			setCount(ref.current + 1);
		}, 1000);
	}
	return (
		<div>
			<p>{count}</p>
			<button onClick={() => setCount(count + 1)}>
				setCount
      </button>
			<button onClick={handleClick}>
				delay setCount
      </button>
		</div>
	)
}

                    # forwardRef

                    函数组件因为没有实例，无法接受ref属性，也就无法实现在函数外部来操作或者获取组件的内部元素。

                    因此需要利用forwardRef来接收并且转发ref属性。

                    import React from 'react';

const FancyButton = React.forwardRef((props, ref) => (
    <button ref={ref} className="FancyButton" onClick={() => {console.log('我被点击了')}}>
      {props.children}
    </button>
  ));

function Main() {
  const ref = React.createRef();
  React.useEffect(() => {
    console.log('ref')
    ref.current && ref.current.click();
  }, [ref]);
  return (
    <FancyButton ref={ref}>Forward Button</FancyButton>
  )
}

                    fowardRef还可以转发HOC的ref属性，从而获取到真正被HOC包裹的组件。

                    function logProps(Component) {

  class LogPropsComponent extends React.Component {
    componentDidUpdate(prevProps) {
      console.log('old props', prevProps);
      console.log('new props', this.props);
    }
    render() {
      // forwardedRef是通过React.forwardRef传过来的。 ‘forwardedRef’名称可以随意
      const {forwardedRef, ...rest} = this.props;
      return <Component ref={forwardedRef} {...rest}></Component>
    }
  }
  // 因为ref不是props，不会被传递下去，如果不这么处理，ref获取到的是高阶组件。
  return React.forwardRef((props, ref) => {
    return <LogPropsComponent {...props} forwardedRef={ref}></LogPropsComponent>
  });
}
class FancyButton extends React.Component {
  focus() {
    console.log('focus');
  }
  render() {
    return (
      <button className="FancyButton" onClick={() => {console.log('我被点击了')}}>
      {this.props.children}
    </button>
    )
  }
}

const LoggedFancyButton = logProps(FancyButton);
function Main() {
  const ref = React.createRef();
  React.useEffect(() => {
    ref.current && ref.current.focus();
  }, [ref]);
  return (
    <LoggedFancyButton ref={ref}>Logged Forward Button</LoggedFancyButton>
  )
}
