    # React Router

    官方文档 v-5

    # 基础用法

    react-router v4以后，将react-router-dom从react-router里拆出来了，需要单独安装并引用react-router-dom。

    配置路由

    v4之前需要在router里指明用到的history类型，也可以指定一个routes对象参数。

    v4以后，BrowserRouter自带browserHistory属性，且==Router下只能有一个子元素。==

    // 之前的用法
// v3
import routes from './routes'
<Router history={browserHistory} routes={routes} />
// or
<Router history={browserHistory}>
  <Route path='/' component={App}>
    // ...
  </Route>
</Router>

    // v4以后的用法 
import { BrowserRouter as Router, Route } from "react-router-dom";
function App() {
  return (
    <Router>
    	<div>
    		<Route path="/about" component={About} />
    		<Route path="/contact" component={Contact} />
  		</div>
    </Router>
  );
}

    switch用途

    switch表示下面的配置中，只有1个路由配置能生效

    // v3 在v3里，可以声明好几个子路由，但是只有第一个被匹配的会被渲染。
<Route path="/" component={App}>
  <IndexRoute component={Home} />
  <Route path="about" component={About} />
  <Route path="contact" component={Contact} />
</Route>

// v4 可以用switch来实现类似的功能。
const App = () => (
  <Switch>
    <Route exact path="/" component={Home} />
    <Route path="/about" component={About} />
    <Route path="/contact" component={Contact} />
  </Switch>
);

    # 动态引入组件

    在项目里经常会需要动态引入组件，而不是一次性把所有组件都加载进来。

    这需要结合动态引入库来使用。 在dva里，有一个dynamic的方法，可以传入路由、组件和models，来进行动态引入。

    import React from "react";
import { Router, Route, Switch, Redirect } from "dva/router";

function RouterConfig({app, history}) {

	// 首页
  const IndexRC = dynamic({
    app,
    models: () => [import("./models/index"), import("./models/ws")],
    component: () => import("./routes/index/index_new"),
  });
  
}

    # 刷新之后嵌套路由找不到

    # 场景描述：

    复杂web应用过程里，需要嵌套路由来实现单页面应用。

    比如，我们先定义了一个路由

     <Route exact path='/' component={Home} />
 <Route exact path='/home' component={Home} />
 <Route exact path='/download' component={DownLoad} />
 <Route exact path='/partnership' component={Partnership} />
 <Route exact path='/my' component={My} />

    然后在Partnership组件里又嵌套了子路由，比如

    <Router>
  <div>
    <SideMenu />
    <div className={classes.content}>
      <Switch>
        <Route exact path="/partnership" component={MyReward}></Route>
        <Route path="/partnership/user_info" component={UserInfo}></Route>
        <Route path="/partnership/my_reward" component={MyReward}></Route>
        <Route
          path="/partnership/affiliate_program"
          component={AffiliateProgram}
          ></Route>
        <Route
          path="/partnership/member_relation"
          component={MemberRelation}
          ></Route>
      </Switch>
    </div>
  </div>
</Router>

    这时候，假如我们/partnership/user_info这个地址上，刷新页面时。会因为找不到路由而跳转到首页。

    # 原因分析：

    因为我们一级路由配置的是精准匹配（exact），所以我们在刷新页面时，Partnership这个组件没有匹配上，导致UserInfo的路由也无法匹配到，所以回到了默认的路由上。

    将上级路由设置成贪婪匹配（不用exact），这样刷新页面时，路由会首先找上级路由，然后再去找它的子路由。
