        # 2.typescript的事件处理

        在ts里，定义函数都需要指定参数的类型，比如常用的click事件，就需要将event指定为一个React.MouseEvent事件。

         const onClick = (event: React.MouseEvent<HTMLButtonElement, MouseEvent>) => {
    if (btnType === "link") {
      const { href, target } = restProps;
      if (href) {
        if (target) {
          window.open(href, target);
        } else {
          window.location.assign(href);
        }
      }
    }
  };

        typescript的Partial函数

        type Partial<T> = { [P in keyof T]?: T[P] };

        keyof得到一个对象的key的枚举。

        in遍历一个枚举对象

        所以Partial可以将一个类型里的变量变为可选的。

        这个在我们定义一个组件时，需要用到，因为组件需要接受参数。

        如果是dom组件，可能还需要接收原生属性，但是原生属性又是非必填的。

        type NativeButtonProps = BaseButtonPrps & ButtonHTMLAttributes<HTMLElement>;
type AnchorButtonProps = BaseButtonPrps & AnchorHTMLAttributes<HTMLElement>;
// 所以用Partial来得到一个可选的属性值
type ButtonProps = Partial<NativeButtonProps & AnchorButtonProps>;

        # 小知识

        npx的作用

        1、它可以避免我们安装一些全局模块。因为一般create-react-app这种脚手架命令我们只是临时用一下，不会说一直要用到。

        npx的原理就是去临时去安装某一个模块到临时文件夹下，用完以后，过一段时间会自动清除掉。 下一次再使用时，如果缓存不存在再去下载。

        2、它还可以帮助我们在命令行里找到并且运行某一个模块

        npx mocha

        比如mocha这个依赖包我们只是当前项目安装了，但是没有安装成全局模块。 直接输入mocha肯定是无法识别的，那么用npx就可以去找到这个依赖工具，并且执行命令。
