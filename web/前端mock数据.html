# 利用webpack来模拟接口

在项目开发里，一般前后端一起开发，前端没有接口可以调用。想要写调用接口，就得用mock的方式来实现。

# 使用框架

结合webpack-api-mocker 和 webpack-dev-server

# 单个mock服务

首先在webpackDevServer.before参数里增加配置：

const apiMocker = require("webpack-api-mocker");

before(app) {
  // 读取mock/index.js文件
  apiMocker(app, path.resolve('../mock/index.js'), {
    changeHost: true,
	}),
}

before里可以拦截所有webpack-dev-server的请求，因此可以用来实现接口的拦截模拟。

我在index.js里模拟了两个接口assets/data、assets/depth，返回了json格式的对象。

const fs = require('fs');

mnodule.exports = {
	'GET /assets/data': (req, res) => {
		return res.json(JSON.parse(fs.readFileSync('./src/assets/data.json', 'utf8')));
	},

	'GET /assets/depth': (req, res) => {
		return res.json(JSON.parse(fs.readFileSync('./src/assets/depth.json', 'utf8')));
	}
}

前端请求这两个路径时，webpack-dev-server就会直接返回json对象。

# 配置通用mock的服务

一般我们需要mock的接口不止一个，尤其是在开发大型项目时，可能需要mock很多接口。

为了让mock文件更清晰，更有条理，我们需要把mock的文件按业务去划分。

为了支持这种修改，我们需要写一个通用的js文件(mock.js)，避免重复写配置。

const fs = require('fs');
const path = require('path')
const mockPath = path.join(__dirname + '/mock');

// 存放文件路径
const mockFiles = [];
fs.readdirSync(mockPath).forEach(file => {
	mockFiles.push(path.resolve('./mock/'+file));
})
console.log(mockFiles);
module.exports=mockFiles;

webpack.config.js

devServer: {
    open: true,
    before(app) {
      // apiMocker(app, path.resolve('./mock/user.js'), {
      //   changeHost: true,
      // })
      apiMocker(app, mockFiles, {
        changeHost: true,
      })
    }
}
